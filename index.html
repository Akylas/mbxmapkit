Raster maps for iOS 7+

A small open source library which adds Mapbox raster map support to apps built with the MapKit framework on iOS 7+.

MBXMapKit{{site.version}}
Installation Supports iOS versions 7.0+
CocoaPods

Follow these steps in order to use the MBXMapKit within your own Xcode 5 project using CocoaPods. This is the easiest installation method if you are familiar with CocoaPods.

  1. Obtain the map ID for your hosted map by clicking Publish and copying the Map ID.
  2. Ensure that your Podfile is targeting :ios and "7.0" or :osx and 10.9.
  3. Add a line containing pod "MBXMapKit" to your Podfile.
  4. Run pod install in the Terminal.
  5. At a minimum, #import <MBXMapKit/MBXMapKit.h> and instantiate an MBXMapView in your code using the -initWithFrame:mapID: initializer and an NSString made from your Map ID obtained above.
Source

Follow these steps in order to use MBXMapKit within your own Xcode 5 project while retaining the ability to edit the library source as you go. This is the most flexible installation method if you want to be able to modify the source code of the library.

  1. Obtain the map ID for your hosted map by clicking Publish and copying the Map ID.
  2. Copy the library source code from the repository.
  3. Add MBXMapKit.h and MBXMapKit.m to your project’s file sources, either by dragging them into your project from Finder or via File > Add Files..., in order to make the library available to your project.
  4. Add MapKit to your app target’s Link Binary With Libraries build phase.
  5. At a minimum, #import MBXMapKit.h and instantiate an MBXMapView in your code using the -initWithFrame:mapID: initializer and an NSString made from your Map ID obtained above.

Why use MBXMapKit?

As part of iOS 7, Apple added the ability to use custom tile overlays with the MapKit framework. MBXMapKit is a lightweight library which leverages MapKit's tile overlay capabilities to help you build awesome apps with Mapbox raster maps. An important thing to understand is that you aren't limited to just picking "Do I want Mapbox Streets or Mapbox Satellite?". The mapbox.com map editor and TileMill provide many options for custom styling and incorporating your own data into a maps hosted by Mapbox.

Here are some examples of what you can do with MBXMapKit:

  • Use a custom styled Mapbox world basemap instead of Apple's basemap. If you've added POI markers to your map with the mapbox.com map editor, MBXMapKit can fetch the marker metadata and icons and prepare custom point annotations for you.
  • Create a transparent regional overlay (i.e. bring your own data) in TileMill, upload it to mapbox.com, then display it as a raster tile overlay on top of the basemap of your choice.
  • Use MBXMapKit's offline map downloader to save regions from Mapbox maps onto your local storage for later use when a reliable network connection is not available.
  • Use MBXMapKit tile overlays in combination with annotations and overlays from other sources, such as your own API. As of version 0.3.0, MBXMapKit is very careful not to disrupt MapKit's normal functionality, so in general, what works with MapKit on its own should also work when you add MBXMapKit to the mix. If you find that isn't the case, [bug reports](https://github.com/mapbox/mbxmapkit/issues/new) are welcome.

API overview

MBXMapKit works identically to Apple’s MapKit, since it’s a subclass derivative, so check out Apple’s MapKit documentation for details on getting started. MBXMapKit provides several additional features:

  • The option to set a mapID to use any Mapbox map.
  • Toggle the visibility of Apple’s default maps under a Mapbox map.
  • Control offline map tile caching duration.
  • Swap in a completely custom offline map to the tile caching system.
  • Support for the Mapbox markers API and Maki.
  • Support for Mapbox-hosted simplestyle GeoJSON.
  • Support for the Mapbox image quality API.

Full API documentation is available right in Xcode, either inline or in Xcode's Quick Help inspector. You can also run the provided install_docs.sh script, which leverages appledoc to create and install a documentation set into Xcode's documentation viewer.

Tile caching

Mapbox offers unrestricted control for caching our maps such as Mapbox Streets on mobile devices with MBXMapKit.

Caching usage
You can cache maps for as much and as long as you like but you may not run a proxy or intermediate cache between Mapbox and devices or furher redistribute maps from a devices cache. For the full details see our Terms of Service.

Retina support

Mapbox Streets has retina support for tiles that can be utilized by MBXMapKit. This brings high-DPI display for devices such as the iPhone 4 and newer, retina iPads, and retina Macs.
