Mapbox to iOS 7 with One Line of Code

A small open source library for building map applications on the Apple platform. Provides highly flexible visual styling & offline use for iPhone, iPad, and Mac devices.

MBXMapKit{{site.version}}
Installation Supports iOS versions 7.0+ & OSX versions 10.9+
CocoaPods

Follow these steps in order to use the MBXMapKit within your own Xcode 5 project using CocoaPods. This is the easiest installation method if you are familiar with CocoaPods.

  1. Obtain the map ID for your hosted map by clicking Publish and copying the Map ID.
  2. Ensure that your Podfile is targeting :ios and "7.0" or :osx and 10.9.
  3. Add a line containing pod "MBXMapKit" to your Podfile.
  4. Run pod install in the Terminal.
  5. At a minimum, #import <MBXMapKit/MBXMapKit.h> and instantiate an MBXMapView in your code using the -initWithFrame:mapID: initializer and an NSString made from your Map ID obtained above.
Source

Follow these steps in order to use MBXMapKit within your own Xcode 5 project while retaining the ability to edit the library source as you go. This is the most flexible installation method if you want to be able to modify the source code of the library.

  1. Obtain the map ID for your hosted map by clicking Publish and copying the Map ID.
  2. Copy the library source code from the repository.
  3. Add MBXMapKit.h and MBXMapKit.m to your project’s file sources, either by dragging them into your project from Finder or via File > Add Files..., in order to make the library available to your project.
  4. Add MapKit to your app target’s Link Binary With Libraries build phase.
  5. At a minimum, #import MBXMapKit.h and instantiate an MBXMapView in your code using the -initWithFrame:mapID: initializer and an NSString made from your Map ID obtained above.

API overview

MBXMapKit works identically to Apple’s MapKit, since it’s a subclass derivative, so check out Apple’s MapKit documentation for details on getting starting. MBXMapKit provides several additional features:

  • The option to set a mapID to use any Mapbox map.
  • Toggle the visibility of Apple’s default maps under a Mapbox map.
  • Control offline map tile caching duration.
  • Swap in a completely custom offline map to the tile caching system.

Tile caching

Mapbox offers unrestricted control for caching our maps such as Mapbox Streets on mobile devices with MBXMapKit.

Caching usage
You can cache maps for as much and as long as you like but you may not run a proxy or intermediate cache between Mapbox and devices or furher redistribute maps from a devices cache. For the full details see our Terms of Service.

Retina support

Mapbox Streets has retina support for tiles that can be utilized by MBXMapKit. This brings high-DPI display for devices such as the iPhone 4 and newer, retina iPads, and retina Macs.
